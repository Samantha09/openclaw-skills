# 安全审查清单

## 1. 注入攻击

### 1.1 SQL 注入
- **检查点**：字符串拼接 SQL 语句
- **风险**：攻击者构造恶意 SQL，读取/篡改/删除数据
- **正确做法**：
  ```java
  // 使用预编译语句（以 Spring JDBC 为例）
  String sql = "SELECT * FROM user WHERE id = ?";
  jdbcTemplate.queryForObject(sql, User.class, userId);
  ```
- **关键词**：`+ sql`、`String.format(sql`、`StringBuilder(sql`

### 1.2 命令注入
- **检查点**：Runtime.exec、ProcessBuilder 拼接用户输入
- **风险**：执行任意系统命令
- **正确做法**：参数列表传递，避免字符串拼接

### 1.3 表达式注入
- **检查点**：SpEL、OGNL、MVEL 解析用户输入
- **风险**：RCE（远程代码执行）
- **正确做法**：表达式沙箱、白名单校验

## 2. 敏感信息泄露

### 2.1 硬编码凭据
- **检查点**：代码中直接出现密码、API Key、私钥
- **关键词**：`password = `、`secret = `、`apiKey = `、`privateKey`
- **正确做法**：使用配置中心、密钥管理服务（KMS）

### 2.2 日志打印敏感信息
- **检查点**：日志中打印密码、身份证号、银行卡号、Token
- **正确做法**：脱敏处理或禁止打印

### 2.3 异常信息泄露
- **检查点**：将详细异常堆栈直接返回给客户端
- **正确做法**：对外返回统一错误码，内部日志记录详细堆栈

---

## 3. 输入校验

### 3.1 缺少白名单校验
- **检查点**：直接接受用户输入并用于业务逻辑
- **正确做法**：
  - 长度限制
  - 类型校验
  - 正则白名单
  - 枚举值校验

### 3.2 文件上传安全
- **检查点**：未限制文件类型、未校验文件内容、路径穿越
- **正确做法**：
  - 白名单后缀
  - 校验 Magic Number
  - 重命名存储，隔离执行
  - 禁止用户控制存储路径

### 3.3 路径遍历
- **检查点**：用户输入拼接文件路径
- **风险**：`../../../etc/passwd`
- **正确做法**：规范化路径后校验是否在允许目录内

---

## 4. 反序列化安全

### 4.1 Java 原生反序列化
- **检查点**：`ObjectInputStream.readObject`
- **风险**：RCE
- **正确做法**：
  - 避免反序列化不可信数据
  - 使用 JSON 等安全格式替代
  - 如需使用，配置白名单类（Look-ahead）

### 4.2 JSON 反序列化
- **检查点**：fastjson、Jackson 等配置不当导致 RCE
- **正确做法**：
  - 及时升级组件版本
  - 关闭 AutoType（fastjson）
  - 使用 `@JsonTypeInfo` 时严格限制子类

---

## 5. 越权与认证

### 5.1 水平越权
- **检查点**：通过修改 ID 参数访问他人数据
- **正确做法**：服务端校验数据归属权

### 5.2 垂直越权
- **检查点**：普通用户调用管理员接口
- **正确做法**：接口和方法级别权限校验

### 5.3 会话安全
- **检查点**：JWT 密钥弱、Token 无过期时间、Session 固定攻击
- **正确做法**：
  - 强密钥（≥256 bits）
  - 合理过期时间
  - HTTPS 传输

---

## 6. SSRF（服务器端请求伪造）

- **检查点**：服务端根据用户输入发起请求
- **风险**：访问内网服务、元数据接口、攻击内部系统
- **正确做法**：
  - 限制目标 URL 协议（仅 http/https）
  - 禁止访问内网 IP 段
  - 白名单域名限制

---

## 7. XSS（跨站脚本）

- **检查点**：用户输入未经转义直接输出到页面
- **风险**：窃取 Cookie、钓鱼、页面篡改
- **正确做法**：
  - 前端/后端输出编码
  - 使用安全模板引擎（自动转义）
  - 富文本使用白名单标签过滤

---

## 8. 依赖与组件安全

### 8.1 已知漏洞组件
- **检查点**：使用了存在已知漏洞的第三方库版本
- **高风险组件**：
  - Log4j 2.x < 2.17.1（Log4Shell / CVE-2021-44228）
  - Fastjson < 1.2.83（AutoType 反序列化 RCE）
  - Jackson < 2.10.x（多态反序列化漏洞）
  - Shiro < 1.10.0（认证绕过）
- **正确做法**：
  - 定期使用 `mvn dependency-check:check` 或 OWASP Dependency-Check 扫描
  - 关注组件安全公告，及时升级
  - 锁定依赖版本（不用 SNAPSHOT）

### 8.2 过度依赖
- **检查点**：引入了功能重复或过大的依赖
- **正确做法**：
  - 只引入需要的模块（如 `guava` 而非 `com.google.collections`）
  - 排除传递依赖中的冲突版本
  - 定期审查未使用的依赖（`mvn dependency:analyze`）

---

## 9. 加密与密钥管理

### 9.1 弱加密算法
- **检查点**：使用 MD5、SHA-1、DES 等弱算法
- **正确做法**：
  - 哈希：SHA-256+ 或 bcrypt/scrypt
  - 对称加密：AES-256
  - 非对称加密：RSA-2048+

### 9.2 硬编码密钥
- **检查点**：密钥写在代码或配置文件中
- **正确做法**：使用密钥管理服务（KMS / Vault / 配置中心加密）
