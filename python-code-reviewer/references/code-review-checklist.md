# 代码审查检查清单

## 命名与格式

- [ ] 类名使用 CapWords
- [ ] 函数/变量名使用 snake_case
- [ ] 常量名使用 UPPER_SNAKE_CASE
- [ ] 文件名使用 lower_with_under.py，不含连字符
- [ ] 行宽不超过 80 字符
- [ ] 4 空格缩进，无 Tab
- [ ] 导入分组正确：标准库 → 第三方 → 本地

## 文档与类型

- [ ] 公开 API 有文档字符串（Google 风格）
- [ ] 文档字符串包含 Args/Returns/Raises（适用时）
- [ ] 公开函数有类型注解
- [ ] 返回值类型注解准确
- [ ] None 使用 `X | None` 而非 `Optional[X]`（3.10+）
- [ ] 泛型填入了类型参数（`dict[str, int]` 而非 `dict`）

## 正确性

- [ ] 无可变默认参数（`def f(x=[])`）
- [ ] 使用 `is None` 而非 `== None`
- [ ] `except:` 不捕获所有异常
- [ ] 循环中不使用 `+` 拼接字符串
- [ ] 资源使用 `with` 语句管理
- [ ] 使用 `if not seq:` 而非 `if len(seq) == 0:`

## 异步与并发

- [ ] async 函数中无阻塞调用（time.sleep、同步 IO）
- [ ] 所有 awaitable 都被 await
- [ ] 线程池有合理配置和生命周期管理
- [ ] 共享状态有同步保护
- [ ] Semaphore / Lock 正确使用
- [ ] asyncio.gather 处理了异常

## 安全

- [ ] 无 eval/exec 处理用户输入
- [ ] SQL 使用参数化查询
- [ ] 无命令注入风险
- [ ] 无硬编码密码/密钥
- [ ] 异常信息不泄露敏感数据
- [ ] 文件路径做了校验

## 设计

- [ ] 函数不超过 40 行
- [ ] 单一职责
- [ ] 无循环依赖
- [ ] 避免全局可变状态
- [ ] 避免深层嵌套（> 3 层）
- [ ] 避免过度使用元类、反射等高级特性

## 测试

- [ ] 有对应的单元测试
- [ ] 覆盖正常和异常路径
- [ ] 测试间相互隔离
- [ ] 单例/全局状态在测试前后重置
